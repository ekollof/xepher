* This Fork

  This is a fork with critical bug fixes applied:

  - *Memory corruption fixes*: Fixed malloc corruption from xmlDocDumpFormatMemory 
    misuse and unaligned allocations that caused crashes
  - *Segfault prevention*: Added comprehensive destructor safety checks to prevent 
    crashes on weechat exit
  - *MAM (Message Archive Management) fixes*:
    - Fixed incorrect JID handling in MAM queries
    - Fixed pagination to properly retrieve all archived messages
    - Fixed time range calculation
    - Enabled automatic MAM retrieval on connect (7 days history for PM channels)
  - *Display improvements*:
    - Messages now show sender's JID instead of client resource ID (e.g., "user@domain.org" 
      instead of "Conversations.xxxx")
    - Typing indicators show JID instead of resource ID
  - *Reduced logging noise*: Disabled debug-level logging for cleaner output
  - *Channel type detection*: Fixed PM vs MUC detection for proper message handling
  - *New features implemented*:
    - MUC leave on /close - properly sends unavailable presence when closing MUC buffer
    - XMPP Ping (XEP-0199) - responds to ping requests and /ping command to send pings
    - Message correction (XEP-0308) - /edit command to correct last sent message
    - Complete chat state notifications - active, composing, paused, inactive, gone
    - PGP key persistence - keys set with /pgp now persist across restarts per JID/MUC

  *Known limitation*: Plugin reload while weechat is running is not supported due to 
  fundamental race conditions with timer hooks. Always restart weechat instead of 
  reloading the plugin.

* Description

  A weechat plugin in C/++ to extend the chat client to
  support XMPP and a currently minimal but ideally maximal
  set of XEPs.
  My priority here is to have an android-available XMPP client
  that hides as little as possible from the user. To use this with
  android, set up a relay (`/relay`) and see weechat-android.
  I am rewriting this in C++, slowly...

* Usage

  1. Start with =/account add <name> <account-name> <password>=, use =/help account= for instructions.

  2. Use =/account connect <account-name>= with the name set at
     add-time.

* Installing

  Place xmpp.so in the appropriate place for weechat plugins.

* Building

  #+begin_src sh
  git clone --depth 1 https://github.com/bqv/weechat-xmpp.git
  cd weechat-xmpp
  git submodule update --init --recursive
  make
  make install
  #+end_src

  Do *NOT* run make install as root, it installs the plugin to your
  local weechat plugins directory

* Development

  I use emacs for development of this, although I am also a fan of vim.
  My debug build process involves static analysis with clang and cppcheck,
  and dynamic analysis with address-sanitizer and leak-sanitizer.
  My debug evaluation process involves gdb/mi run with the arguments
  =-ex "handle SIGPIPE nostop noprint pass" --args weechat -a 2>asan.log=
  since part of weechat and it's default plugins use SIGPIPE as control.

  I don't expect any, but I have no real requests for style of pull requests
  besides a wish that you keep vaguely to the indentation style I use.

* This Fork

  This is a fork with critical bug fixes applied:

  - *Memory corruption fixes*: Fixed malloc corruption from xmlDocDumpFormatMemory
    misuse and unaligned allocations that caused crashes
  - *Segfault prevention*: Added comprehensive destructor safety checks to prevent
    crashes on weechat exit
  - *WeeChat 4.3.0+ compatibility*: Updated base64 API calls for compatibility with
    newer WeeChat versions while maintaining backwards compatibility
  - *MAM (Message Archive Management) fixes*:
    - Fixed MAM IQ routing: PM queries were sent to the contact's JID instead
      of the local server domain, so the server discarded them
    - Fixed UTC timestamp parsing: =strptime= fills a UTC =struct tm= but
      =mktime= was interpreting it as local time (CET = UTC+1), shifting all
      timestamps one hour into the future
    - Fixed multi-page MAM results: the global discovery query stalled after
      the first RSM page and never fetched newer messages; it now pages forward
      until the archive is exhausted
    - Fixed single-page / empty =<fin>= handling: when a server sends
      =<fin complete='true'/>= with no =<set>= child the query was never
      cleaned up and the fetch timestamp was never updated
    - Fixed channel re-open suppression: a =-1= sentinel in LMDB that
      prevented auto-recreation of closed buffers was not cleared on re-open,
      so MAM never ran for those channels
    - Enabled automatic MAM retrieval on connect (7 days history for PM channels)
    - Full message caching with LMDB — instant display from cache, then fetch new ones
  - *Display improvements*:
    - Messages now show sender's JID instead of client resource ID (e.g., "user@domain.org"
      instead of "Conversations.xxxx")
    - Typing indicators show bare JID instead of resource ID (both in buffer and bar)
    - Encryption status indicator - visual indicator in status bar showing OMEMO/PGP/plaintext
    - Message styling (XEP-0393) - renders *bold*, _italic_, `code`, ~strikethrough~, quotes
  - *Reduced logging noise*: Disabled debug-level logging for cleaner output
  - *Channel type detection*: Fixed PM vs MUC detection for proper message handling
   - *New features implemented*:

    - MUC leave on /close - properly sends unavailable presence when closing MUC buffer

    - Message correction (XEP-0308) - /edit command to correct last sent message

* Description

  A weechat plugin in C/++ to extend the chat client to
  support XMPP and a currently minimal but ideally maximal
  set of XEPs.
  My priority here is to have an android-available XMPP client
  that hides as little as possible from the user. To use this with
  android, set up a relay (`/relay`) and see weechat-android.
  I am rewriting this in C++, slowly...

* Usage

  1. Start with =/account add <name> <account-name> <password>=, use =/help account= for instructions.

  2. Use =/account connect <account-name>= with the name set at
     add-time.

** Encryption Status Indicator

   The plugin provides a bar item =xmpp_encryption= that displays the current
   encryption status of the active buffer. It will show:
   - =🔒OMEMO= when OMEMO encryption is enabled
   - =🔒PGP= when PGP encryption is enabled
   - Empty when using plain text

   To add it to your status bar:
   #+begin_src
   /set weechat.bar.status.items "[time],[buffer_last_number],[buffer_plugin],buffer_number+:+buffer_name+(buffer_modes)+{buffer_nicklist_count}+buffer_zoom+buffer_filter,xmpp_encryption,[lag],[hotlist],completion"
   #+end_src

   Or simply add =xmpp_encryption= to your existing status bar items configuration.

** Typing Indicator

   The plugin sends and receives XEP-0085 chat state notifications (typing
   indicators). To display them you need to configure both WeeChat and your
   status bar.

*** 1. Enable typing in WeeChat

   #+begin_src
   /set typing.look.enabled_nicks on
   /set typing.look.enabled_self on
   /save typing
   #+end_src

   - =enabled_nicks= — display when contacts are typing
   - =enabled_self= — send typing notifications when you type
   - =input_min_chars= (default 4) — how many characters you must type before
     WeeChat considers you "composing"; lower it if you want more responsive
     outgoing notifications:
     #+begin_src
     /set typing.look.input_min_chars 2
     #+end_src

*** 2. Add the typing bar item

   WeeChat ships a dedicated =typing= bar by default. Its item must be set to
   the built-in =typing= item name (not =xmpp_typing=):

   #+begin_src
   /set weechat.bar.typing.items "typing"
   /save
   #+end_src

   The bar already has =conditions = "${typing}"= so it only appears when
   someone is actually typing — no permanent screen real estate is used.

** Message Status Glyphs

   Sent messages show an inline delivery/read status glyph that updates
   automatically:

   | Glyph | Meaning                                   |
   |-------|-------------------------------------------|
   | ⌛    | Pending — message sent, no receipt yet    |
   | ✓     | Delivered — recipient's client received it (XEP-0184) |
   | ✓✓   | Read — recipient has marked it displayed (XEP-0333)   |

   No configuration required; glyphs appear automatically on outgoing messages.

** XHTML-IM Rendering (XEP-0071)

   When a message contains an =<html>= body (XEP-0071: XHTML-IM), the plugin
   renders it with full WeeChat color and attribute support instead of showing
   raw markup. The XHTML body replaces the plain-text body entirely (unless the
   message is encrypted, in which case the plain body is used as a fallback).

   Supported elements and their rendering:

   | Element                          | Rendered as                              |
   |----------------------------------|------------------------------------------|
   | =<b>=, =<strong>=                | Bold                                     |
   | =<i>=, =<em>=                    | Italic                                   |
   | =<u>=                            | Underline                                |
   | =<del>=, =<s>=, =<strike>=       | Darkgray (strikethrough approximation)   |
   | =<code>=, =<tt>=                 | Gray monospace                           |
   | =<pre>=                          | Gray-wrapped preformatted block          |
   | =<a href="...">text</a>=         | =text (blue URL)= or just blue URL       |
   | =<img alt="..."/>=               | =[alt]= placeholder                      |
   | =<br>=                           | Newline                                  |
   | =<p>=, =<div>=, =<li>=           | Block with surrounding newlines          |
   | =<blockquote>=                   | Green => = prefix on every line          |
   | =<span style="color:...">=       | Terminal color (16 named + #rgb hex)     |
   | =<span style="font-weight:bold">= | Bold                                    |
   | =<span style="font-style:italic">= | Italic                                 |
   | Unknown/container tags           | Transparent pass-through                 |

   Background colors in CSS =style= attributes are intentionally ignored to
   avoid unreadable output on dark or light terminal themes.

** Ad-Hoc Commands and Data Forms (XEP-0050 / XEP-0004)

   The =/adhoc= command lets you execute server-side commands directly from
   WeeChat. When a command returns a data form (XEP-0004), the form is rendered
   inline in the current buffer.

*** Reading a rendered form

   A rendered form looks like:

   #+begin_src
   ── Ad-Hoc Form: Announce ──
   Instructions: Fill in the announcement details.

    1. Subject* [subject/text-single] = (empty)
    2. Body*    [body/text-multi]     = (empty)
    3. Urgent   [urgent/boolean]      = false
   ──────────────────────────────────────────────
   Submit: /adhoc example.com announce <sessionid> subject=... body=... urgent=...
   #+end_src

   - Fields marked with =*= are required.
   - =list-single=/=list-multi= fields show all options with the current
     selection highlighted in green.
   - =text-private= fields display as =********=.
   - =hidden= fields are silently omitted.

*** Submitting a form

   Copy the submit hint at the bottom of the form and fill in your values:

   #+begin_src
   /adhoc example.com announce <sessionid> subject=Hello body=World urgent=true
   #+end_src

   For multi-line values, use a space-separated list of =field=value= pairs.
   Multi-step sessions are supported; each response includes the session ID for
   the next step.

    #+end_src

    #+begin_src

*** /retract
    Retract (delete) the last message sent (XEP-0424: Message Retraction)

    *Usage:*
    #+begin_src
    /retract
    #+end_src

    *Example:*
    #+begin_src
    /retract
    #+end_src

    *Note:* This sends a retraction request to all recipients. The message will
    be replaced with a tombstone "[Message deleted]" in the buffer. Note that
    recipients may have already seen the message before retraction. Works in both
    PM and MUC channels.

*** /moderate
    Moderate (retract) the last message in a MUC (XEP-0425: Message Moderation)

    *Usage:*
    #+begin_src
    /moderate [reason]
    #+end_src

    *Examples:*
    #+begin_src
    /moderate
    /moderate Spam message
    /moderate Violates community guidelines
    #+end_src

    *Note:* This command is for MUC moderators to remove messages from other users.
    Moderated messages are replaced with a tombstone showing the moderation reason.
    Use /retract to delete your own messages. Only works in MUC rooms where you
    have moderator privileges. The server must support XEP-0425.

*** /react
    Send an emoji reaction to the last received message (XEP-0444: Message Reactions)

    *Usage:*
    #+begin_src
    /react <emoji>
    #+end_src

    *Examples:*
    #+begin_src
    /react 👍
    /react ❤️
    /react 😂
    #+end_src

    *Note:* Reacts to the last message from another user in the current buffer.
    Reactions are displayed appended to the original message in blue brackets.
    Works in both PM and MUC channels. The server must support XEP-0444.

*** /reply
    Reply to the last received message with context (XEP-0461: Message Replies)

    *Usage:*
    #+begin_src
    /reply <message>
    #+end_src

    *Examples:*
    #+begin_src
    /reply Thanks for the info!
    /reply I agree with that
    /reply Great idea, let's do it
    #+end_src

    *Note:* Replies to the last message from another user in the current buffer.
    The reply includes a reference to the original message, displayed with a
    cyan arrow (↪) and excerpt of the original message. Works in both PM and
    MUC channels. The server must support XEP-0461.

*** /whois
    Get vCard information about an XMPP user (XEP-0054: vcard-temp)

    #+begin_src

    #+end_src

    *Examples:*

** Encryption

*** /omemo
    Manage OMEMO encryption for the current buffer

    *Usage:*
    #+begin_src
    /omemo [check | republish | status | reset-keys]
    #+end_src

    *Subcommands:*
    - (no args): Enable OMEMO for current channel
    - =check=: Verify OMEMO bundle is published on server
    - =republish=: Republish bundle (fixes missing device keys)
    - =status=: Show OMEMO status and device ID
    - =reset-keys=: Reset OMEMO key database to force key renegotiation

    *Examples:*
    #+begin_src
    /omemo                    # Enable OMEMO for current channel
    /omemo check              # Check bundle on server
    /omemo republish          # Republish bundle
    /omemo status             # Show device ID and status
    /omemo reset-keys         # Reset keys (forces renegotiation)
    #+end_src

    *Note:* OMEMO provides end-to-end encryption. Keys are automatically
    generated and managed by the plugin. Use =reset-keys= if you're having
    key exchange issues.

*** /pgp
    Manage PGP encryption for the current buffer

    *Usage:*
    #+begin_src
    /pgp [<keyid> | status | reset]
    #+end_src

    *Subcommands:*
    - (no args): Enable PGP for current channel (use existing keys)
    - =<keyid>=: Add or enable PGP with specific key ID
    - =status=: Show configured PGP keys for current channel
    - =reset=: Remove all PGP keys from current channel

    *Examples:*
    #+begin_src
    /pgp                      # Enable PGP with existing keys
    /pgp A1B2C3D4E5F6         # Add key to channel
    /pgp status               # Show which keys are configured
    /pgp reset                # Remove all keys from channel
    #+end_src

    *Note:* PGP key settings persist across restarts per JID/MUC. Uses
    your local GPG keyring.

*** /plain
    Disable encryption for the current buffer (use plaintext)

    *Usage:*
    #+begin_src
    /plain
    #+end_src

** File Sharing

*** /upload
    Upload a file via HTTP File Upload (XEP-0363)

    *Usage:*
    #+begin_src
    /upload [<filename>]
    #+end_src

    *Examples:*
    #+begin_src
    /upload                      # Interactive file picker
    /upload /path/to/image.jpg   # Direct path
    /upload ~/Documents/file.pdf
    /upload screenshot.png
    #+end_src

    *Interactive File Picker:* When called without arguments, ==/upload== automatically
    launches an interactive file picker:
    - Detects X11/Wayland display servers via =$DISPLAY= and =$WAYLAND_DISPLAY=
    - Tries GUI file dialogs: ==zenity== (GTK) or ==kdialog== (KDE)
    - Falls back to ==fzf== in terminal if no GUI available
    - Shows helpful error if no picker tools are installed

    *Note:* The file will be uploaded to the server's HTTP upload service
    and a URL will be automatically sent in the chat with XEP-0066 Out of Band Data.
    The server must support XEP-0363 (HTTP File Upload). File size limits are
    determined by the server.

*** /adhoc
    Execute server-side ad-hoc commands (XEP-0050)

    *Usage:*
    #+begin_src
    /adhoc <jid>
    /adhoc <jid> <node>
    /adhoc <jid> <node> <sessionid> [field=value ...]
    #+end_src

    *Examples:*
    #+begin_src
    /adhoc example.com                           # List available commands on the server
    /adhoc example.com announce                  # Execute the 'announce' command
    /adhoc example.com config <sessionid> subject=Hello body=World  # Submit a form
    #+end_src

    *Workflow:*

    1. Run =/adhoc <jid>= to see a list of available commands (nodes).
    2. Run =/adhoc <jid> <node>= to execute a command. If it returns a data form,
       the form is rendered inline in the current buffer with field names, types,
       current values, and a submit hint.
    3. Copy the submit hint and fill in field values, then send it as the next
       command: =/adhoc <jid> <node> <sessionid> field1=value field2=value=

    *Note:* Multi-step sessions are supported. Each form response includes the
    session ID needed for the next submission step.

*** /bookmark
    Manage XMPP bookmarks (XEP-0048: Bookmark Storage)

    *Usage:*
    #+begin_src
    /bookmark
    /bookmark add [jid] [name]
    /bookmark del <jid>
    /bookmark autojoin <jid> <on|off>
    #+end_src

    *Examples:*
    #+begin_src
    /bookmark                                      # List all bookmarks
    /bookmark add                                  # Bookmark current MUC
    /bookmark add room@conference.example.com      # Bookmark specific MUC
    /bookmark add room@conference.example.com My Room
    /bookmark del room@conference.example.com      # Remove bookmark
    /bookmark autojoin room@conference.example.com on
    #+end_src

    *Note:* Bookmarks are stored on the server and synchronized across
    all clients. Rooms with autojoin enabled will be automatically joined
    on connect, except for IRC gateway rooms (biboumi) which are skipped
    to prevent connection issues. These are detected by the presence of
    =%=, =biboumi=, or =@irc.= in the JID. Aliases =delete= and =remove=
    can be used instead of =del=.

** WeeChat Version Compatibility

   *Important*: WeeChat 4.3.0 (May 2024) introduced an API change for base64
   encoding/decoding functions. This fork is currently configured for WeeChat >= 4.3.0.

   If you're using an older WeeChat version (< 4.3.0) and get compilation errors
   about =invalid conversion from 'int' to 'const char*'=, you need to adjust the
   API compatibility setting:

   1. Open =omemo.cpp=
   2. Find the section around line 42:
      #+begin_src cpp
      #if 1  // Set to 0 for WeeChat < 4.3.0
          #define WEECHAT_BASE64 "64"
      #else
          #define WEECHAT_BASE64 64
      #endif
      #+end_src
   3. Change =#if 1= to =#if 0= for WeeChat < 4.3.0
   4. Rebuild with =make clean && make=

   To check your WeeChat version: =weechat --version=

* Building

  #+begin_src sh
  git clone --depth 1 https://github.com/bqv/weechat-xmpp.git
  cd weechat-xmpp
  make install-deps              # Install system dependencies (requires sudo)
  git submodule update --init --recursive
  make
  make install
  #+end_src

  The =make install-deps= target automatically detects your Linux distribution
  and installs required dependencies (libstrophe, g++, etc.). Supported distros:
  Debian/Ubuntu, Fedora/RHEL, Arch, openSUSE, Void, Alpine, Gentoo.

  Do *NOT* run make install as root, it installs the plugin to your
  local weechat plugins directory

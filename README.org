* This Fork

  This is a fork with critical bug fixes applied:

  - *Memory corruption fixes*: Fixed malloc corruption from xmlDocDumpFormatMemory 
    misuse and unaligned allocations that caused crashes
  - *Segfault prevention*: Added comprehensive destructor safety checks to prevent 
    crashes on weechat exit
  - *WeeChat 4.3.0+ compatibility*: Updated base64 API calls for compatibility with
    newer WeeChat versions while maintaining backwards compatibility
  - *MAM (Message Archive Management) fixes*:
    - Fixed incorrect JID handling in MAM queries
    - Fixed pagination to properly retrieve all archived messages
    - Fixed time range calculation
    - Enabled automatic MAM retrieval on connect (7 days history for PM channels)
    - Full message caching with LMDB - instant message display from cache, then fetch new ones
  - *Display improvements*:
    - Messages now show sender's JID instead of client resource ID (e.g., "user@domain.org" 
      instead of "Conversations.xxxx")
    - Typing indicators show bare JID instead of resource ID (both in buffer and bar)
    - Encryption status indicator - visual indicator in status bar showing OMEMO/PGP/plaintext
    - Message styling (XEP-0393) - renders *bold*, _italic_, `code`, ~strikethrough~, quotes
  - *Reduced logging noise*: Disabled debug-level logging for cleaner output
  - *Channel type detection*: Fixed PM vs MUC detection for proper message handling
  - *New features implemented*:
    - Stream Management (XEP-0198) - reliable message delivery and session resumption
    - Blocking Command (XEP-0191) - /block, /unblock, /blocklist for privacy
    - MUC Self-Ping (XEP-0410) - /selfping to verify room membership
    - Personal Eventing Protocol (XEP-0163) - foundation for avatars and modern features
    - Direct MUC Invitations (XEP-0249) - /invite command to invite users to rooms
    - The /me Command (XEP-0245) - send IRC-style action messages
    - vcard-temp (XEP-0054) - /whois command to retrieve user information
    - MUC leave on /close - properly sends unavailable presence when closing MUC buffer
    - XMPP Ping (XEP-0199) - responds to ping requests and /ping command to send pings
    - Message correction (XEP-0308) - /edit command to correct last sent message
    - Message retraction (XEP-0424) - /retract command to delete last sent message with MAM support
    - Message styling (XEP-0393) - automatic rendering of formatted text
    - Complete chat state notifications - active, composing, paused, inactive, gone
    - PGP key persistence - keys set with /pgp now persist across restarts per JID/MUC
    - Encryption status bar item (xmpp_encryption) - shows current buffer's encryption mode
    - Roster management - /roster command to display, add, and remove contacts
    - Bookmark management - /bookmark command to add, delete, list, and toggle autojoin for MUC rooms
    - Room explorer - /list command to search public MUC rooms via search.jabber.network
    - Entity capability caching (XEP-0115) - caches disco#info results to reduce network queries

  See the *XMPP Compliance* section below for a comprehensive list of implemented XEPs.

  *Known limitation*: Plugin reload while weechat is running is not supported due to 
  fundamental race conditions with timer hooks. Always restart weechat instead of 
  reloading the plugin.

* Description

  A weechat plugin in C/++ to extend the chat client to
  support XMPP and a currently minimal but ideally maximal
  set of XEPs.
  My priority here is to have an android-available XMPP client
  that hides as little as possible from the user. To use this with
  android, set up a relay (`/relay`) and see weechat-android.
  I am rewriting this in C++, slowly...

* Usage

  1. Start with =/account add <name> <account-name> <password>=, use =/help account= for instructions.

  2. Use =/account connect <account-name>= with the name set at
     add-time.

** Encryption Status Indicator

   The plugin provides a bar item =xmpp_encryption= that displays the current
   encryption status of the active buffer. It will show:
   - =🔒OMEMO= when OMEMO encryption is enabled
   - =🔒PGP= when PGP encryption is enabled
   - Empty when using plain text

   To add it to your status bar:
   #+begin_src
   /set weechat.bar.status.items "[time],[buffer_last_number],[buffer_plugin],buffer_number+:+buffer_name+(buffer_modes)+{buffer_nicklist_count}+buffer_zoom+buffer_filter,xmpp_encryption,[lag],[hotlist],completion"
   #+end_src

   Or simply add =xmpp_encryption= to your existing status bar items configuration.

** Roster Management

   The plugin supports XMPP roster (contact list) management:
   - =/roster= - Display your contact list with names, subscriptions, and groups
   - =/roster add <jid> [name]= - Add a contact to your roster and send subscription request
   - =/roster del <jid>= - Remove a contact from your roster

   The roster is automatically fetched when you connect to the server.

* Commands

  The plugin provides the following commands:

** Account Management

*** /account
    Handle XMPP accounts

    *Usage:*
    #+begin_src
    /account list
    /account add <name> <jid> <password>
    /account connect <account>
    /account disconnect <account>
    /account reconnect <account>
    /account delete <account>
    #+end_src

    *Examples:*
    #+begin_src
    /account add work user@example.com mypassword
    /account connect work
    /account list
    /account disconnect work
    /account delete work
    #+end_src

** Communication

*** /enter
    Enter an XMPP multi-user chat (MUC)

    *Usage:*
    #+begin_src
    /enter <jid>
    #+end_src

    *Example:*
    #+begin_src
    /enter room@conference.example.com
    #+end_src

*** /open
    Open a direct XMPP chat (private message)

    *Usage:*
    #+begin_src
    /open <jid>
    #+end_src

    *Example:*
    #+begin_src
    /open friend@example.com
    #+end_src

*** /msg
    Send an XMPP message to the current buffer

    *Usage:*
    #+begin_src
    /msg <message>
    #+end_src

    *Example:*
    #+begin_src
    /msg Hello, how are you?
    #+end_src

*** /me
    Send an XMPP action to the current buffer (third-person narrative)

    Implements XEP-0245: The /me Command

    *Usage:*
    #+begin_src
    /me <message>
    #+end_src

    *Example:*
    #+begin_src
    /me is away from keyboard
    /me jumps for joy
    #+end_src

*** /invite
    Invite a user to join the current MUC room (XEP-0249: Direct MUC Invitations)

    *Usage:*
    #+begin_src
    /invite <jid> [<reason>]
    #+end_src

    *Examples:*
    #+begin_src
    /invite friend@example.com
    /invite colleague@example.com Join us for the meeting!
    #+end_src

    *Note:* Can only be used in MUC (room) buffers. The invited user will 
    receive a notification with instructions to join.

*** /selfping
    Verify MUC membership with self-ping (XEP-0410: MUC Self-Ping)

    *Usage:*
    #+begin_src
    /selfping
    #+end_src

    *Example:*
    #+begin_src
    /selfping (in a MUC room buffer)
    #+end_src

    *Note:* Sends a ping to your own MUC nickname to verify you are still 
    joined to the room. Useful after network interruptions or to detect 
    silent kicks/disconnects. Can only be used in MUC (room) buffers.

*** /edit
    Edit the last message sent (XEP-0308: Message Correction)

    *Usage:*
    #+begin_src
    /edit <message>
    #+end_src

    *Example:*
    #+begin_src
    /edit I meant to say something else!
    #+end_src

    *Note:* The original message will be updated with a visual indicator 
    showing the diff between the old and new text.

*** /retract
    Retract (delete) the last message sent (XEP-0424: Message Retraction)

    *Usage:*
    #+begin_src
    /retract
    #+end_src

    *Example:*
    #+begin_src
    /retract
    #+end_src

    *Note:* This sends a retraction request to all recipients. The message will 
    be replaced with a tombstone "[Message deleted]" in the buffer. Note that 
    recipients may have already seen the message before retraction. Works in both 
    PM and MUC channels.

*** /whois
    Get vCard information about an XMPP user (XEP-0054: vcard-temp)

    *Usage:*
    #+begin_src
    /whois [<jid>]
    #+end_src

    *Examples:*
    #+begin_src
    /whois friend@example.com
    /whois (in a PM buffer - uses the current contact)
    #+end_src

    *Note:* Displays available vCard information including full name, 
    nickname, email, URL, and description if provided by the user.

** Privacy & Blocking

*** /block
    Block one or more JIDs (XEP-0191: Blocking Command)

    *Usage:*
    #+begin_src
    /block <jid> [<jid>...]
    #+end_src

    *Examples:*
    #+begin_src
    /block spam@example.com
    /block user1@server.com user2@server.com
    #+end_src

    *Note:* Blocked users cannot send you messages or presence updates.

*** /unblock
    Unblock one or more JIDs, or all blocked JIDs (XEP-0191: Blocking Command)

    *Usage:*
    #+begin_src
    /unblock [<jid> [<jid>...]]
    #+end_src

    *Examples:*
    #+begin_src
    /unblock spam@example.com
    /unblock (unblocks all JIDs)
    #+end_src

*** /blocklist
    Display all currently blocked JIDs (XEP-0191: Blocking Command)

    *Usage:*
    #+begin_src
    /blocklist
    #+end_src

** Encryption

*** /omemo
    Enable OMEMO encryption for the current buffer

    *Usage:*
    #+begin_src
    /omemo
    #+end_src

    *Note:* OMEMO provides end-to-end encryption. Keys are automatically 
    generated and managed by the plugin.

*** /pgp
    Enable PGP encryption for the current buffer

    *Usage:*
    #+begin_src
    /pgp <keyid>
    #+end_src

    *Example:*
    #+begin_src
    /pgp A1B2C3D4E5F6
    #+end_src

    *Note:* The PGP key setting persists across restarts for the current 
    JID/MUC. Uses your GPG keyring.

*** /plain
    Disable encryption for the current buffer (use plaintext)

    *Usage:*
    #+begin_src
    /plain
    #+end_src

** History & Archive

*** /mam
    Retrieve archived messages using MAM (XEP-0313: Message Archive Management)

    *Usage:*
    #+begin_src
    /mam [days]
    #+end_src

    *Examples:*
    #+begin_src
    /mam        # Retrieve last 7 days by default
    /mam 30     # Retrieve last 30 days
    #+end_src

    *Note:* Messages are cached locally using LMDB and retrieved automatically 
    on connect for PM channels.

** Service Discovery

*** /disco
    Discover XMPP services and features (XEP-0030: Service Discovery)

    *Usage:*
    #+begin_src
    /disco [jid]
    #+end_src

    *Examples:*
    #+begin_src
    /disco                           # Discover server features
    /disco conference.example.com    # Discover MUC service
    #+end_src

*** /roster
    Manage XMPP roster (contact list)

    *Usage:*
    #+begin_src
    /roster
    /roster add <jid> [name]
    /roster del <jid>
    #+end_src

    *Examples:*
    #+begin_src
    /roster                              # Display contact list
    /roster add friend@example.com John  # Add contact with name
    /roster del friend@example.com       # Remove contact
    #+end_src

    *Note:* Aliases =delete= and =remove= can be used instead of =del=.

*** /list
    Search for public MUC rooms using the search.jabber.network directory

    *Usage:*
    #+begin_src
    /list [keywords]
    #+end_src

    *Examples:*
    #+begin_src
    /list                    # List popular public rooms
    /list xmpp               # Search for rooms about XMPP
    /list linux gaming       # Search for linux gaming rooms
    #+end_src

    *Note:* This command queries the public room directory at 
    search.jabber.network. Results show room address, name, user count, 
    and description. Use =/enter <address>= to join a room from the list.

*** /bookmark
    Manage XMPP bookmarks (XEP-0048: Bookmark Storage)

    *Usage:*
    #+begin_src
    /bookmark
    /bookmark add [jid] [name]
    /bookmark del <jid>
    /bookmark autojoin <jid> <on|off>
    #+end_src

    *Examples:*
    #+begin_src
    /bookmark                                      # List all bookmarks
    /bookmark add                                  # Bookmark current MUC
    /bookmark add room@conference.example.com      # Bookmark specific MUC
    /bookmark add room@conference.example.com My Room
    /bookmark del room@conference.example.com      # Remove bookmark
    /bookmark autojoin room@conference.example.com on
    #+end_src

    *Note:* Bookmarks are stored on the server and synchronized across 
    all clients. Rooms with autojoin enabled will be automatically joined 
    on connect, except for IRC gateway rooms (biboumi) which are skipped 
    to prevent connection issues. These are detected by the presence of 
    =%=, =biboumi=, or =@irc.= in the JID. Aliases =delete= and =remove= 
    can be used instead of =del=.

** WeeChat Version Compatibility

   *Important*: WeeChat 4.3.0 (May 2024) introduced an API change for base64
   encoding/decoding functions. This fork is currently configured for WeeChat >= 4.3.0.

   If you're using an older WeeChat version (< 4.3.0) and get compilation errors 
   about =invalid conversion from 'int' to 'const char*'=, you need to adjust the 
   API compatibility setting:

   1. Open =omemo.cpp=
   2. Find the section around line 42:
      #+begin_src cpp
      #if 1  // Set to 0 for WeeChat < 4.3.0
          #define WEECHAT_BASE64 "64"
      #else
          #define WEECHAT_BASE64 64
      #endif
      #+end_src
   3. Change =#if 1= to =#if 0= for WeeChat < 4.3.0
   4. Rebuild with =make clean && make=

   To check your WeeChat version: =weechat --version=

* Building

  #+begin_src sh
  git clone --depth 1 https://github.com/bqv/weechat-xmpp.git
  cd weechat-xmpp
  make install-deps              # Install system dependencies (requires sudo)
  git submodule update --init --recursive
  make
  make install
  #+end_src

  The =make install-deps= target automatically detects your Linux distribution
  and installs required dependencies (libstrophe, g++, etc.). Supported distros:
  Debian/Ubuntu, Fedora/RHEL, Arch, openSUSE, Void, Alpine, Gentoo.

  Do *NOT* run make install as root, it installs the plugin to your
  local weechat plugins directory
